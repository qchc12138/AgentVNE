## 一、整体架构

### 1.1 架构层次

```

                     │

                     │
```

## 三、核心组件详解

### 3.1 策略协议 (PlacementStrategy)

**位置**：`tests/test_strategy.py`

**接口定义**：
```python
@runtime_checkable
class PlacementStrategy(Protocol):
    name: str
    
    def prepare(self, env: SimuVNEEnv) -> None:
        """策略执行前的准备逻辑（如加载模型、初始化参数）。"""
    
    def place(
        self,
        vn: Data,              # 虚拟网络图
        sn_state: Data,        # 物理网络状态
        env: SimuVNEEnv,       # 环境对象（只读）
        *,
        context: StrategyContext,  # 上下文信息
    ) -> StrategyResult:
        """对单个 VN 任务执行放置决策。"""
```

**关键约束**：
- 策略只能**读取**环境，不能直接修改主环境的资源
- 资源扣减由测试框架统一管理
- 策略在环境副本上执行搜索和资源扣减

### 3.2 测试执行器 (SingleTester)

**位置**：`tests/test_strategy.py`

**核心方法**：

1. **`prepare(strategy)`**：初始化测试环境
   - 创建 `SimuVNEEnv`
   - 创建 `WorkflowGenerator`
   - 调用 `strategy.prepare(env)`

2. **`run()`**：执行测试流程
   - 时间驱动的主循环
   - 任务到达检测（泊松分布）
   - 调用策略进行放置决策
   - 处理结果（成功/失败）
   - 资源扣减与回滚
   - 统计信息收集

### 3.3 配置管理

**位置**：`tests/test_configs.py`

**TestConfig**：单次测试的完整配置
```python
@dataclass
class TestConfig:
    sn_topology_path: str      # SN拓扑文件路径
    workflow_types: Dict[str, str]  # 工作流类型映射
    arrival_rate: float         # 任务到达率（泊松分布）
    mean_lifetime: float        # 平均生存时间（指数分布）
    max_time_steps: int         # 最大时间步数
    device: str                 # 运行设备
    seed: int                   # 随机种子
    penalty: float              # 放置失败惩罚
```

**ParameterSpec**：参数规范
```python
@dataclass
class ParameterSpec:
    arrival_rate: float
    mean_lifetime: float
    max_time_steps: int
    seed: int
    device: Optional[str] = None
    penalty: Optional[float] = None
```

### 3.4 输出管理 (TestPrinter)

**位置**：`tests/test_printer.py`

**功能**：
- 结果表格打印
- 日志文件管理
- 可视化图表生成
- 详细日志记录（时间步级别）
- 参数扫描结果汇总

**输出目录**：`tests/outs/`
- `tester_outs/` - 多策略对比测试
- `tester_sweep_outs/` - 参数扫描测试
- `*_single_outs/` - 单策略测试

### 4.1 添加新策略

#### 步骤 1：实现策略类

创建 `tests/tester_xxx.py`：

```python
from tests.test_strategy import PlacementStrategy, StrategyResult, StrategyContext
from torch_geometric.data import Data
from env import SimuVNEEnv
import copy

class XxxPlacementStrategy(PlacementStrategy):
    name: str = "xxx"
    
    def __init__(self, param1: str, param2: int = 10):
        self.param1 = param1
        self.param2 = param2
    
    def prepare(self, env: SimuVNEEnv) -> None:
        # 初始化逻辑（如加载模型、初始化参数）
        pass
    
    def place(
        self,
        vn: Data,
        sn_state: Data,
        env: SimuVNEEnv,
        *,
        context: StrategyContext,
    ) -> StrategyResult:
        # 在环境副本上执行搜索
        sim_env = copy.deepcopy(env)
        mapping = self._search(vn, sim_env)
        success = len(mapping) == vn.x.size(0)
        return StrategyResult(
            success=success,
            mapping=mapping if success else {},
            metadata={"param1": self.param1}
        )
```

## 八、常见问题

## 九、最佳实践

1. **策略实现**：
   - 始终在环境副本上执行搜索
   - 不要直接修改主环境
   - 返回完整的映射结果

2. **配置管理**：
   - 使用 `ParameterSpec` 管理参数
   - 通过命令行参数或代码配置灵活切换

3. **输出管理**：
   - 使用 `TestPrinter` 统一管理输出
   - 保持输出格式一致性

4. **错误处理**：
   - 在策略中提供清晰的错误信息
   - 使用 `metadata` 记录调试信息

5. **测试验证**：
   - 先运行单策略测试验证功能
   - 再运行多策略对比测试
