## 测试框架与策略概览

本文总结 `tests/` 目录及 `tester.py` 的整体设计、使用方式与各策略实现思路，便于后续扩展。

### 1. 代码结构

- `test_strategy.py`：核心执行单元  
  - `PlacementStrategy` 协议：策略只需实现 `prepare`/`place`，禁止直接修改 `env`。  
  - `SingleTester`、`run_single_strategy_test`、`run_strategy_with_details`：统一初始化环境、调度任务、扣减资源、汇总指标。
- `test_printer.py`：日志/绘图/输出目录管理  
  - `TestPrinter` 负责 session 目录、表格与图表。  
  - `DetailedTestLogger` 记录时间步、资源快照、路径等细节。  
  - `TEST_SCOPE_DIRS` 将不同测试（如 `null_single`、`tester`）映射到 `tests/outs/` 子目录。
- `test_configs.py`：配置与参数集  
  - `TestConfig`、`ParameterSpec`、`parse_parameters` 等帮助函数。  
  - `SMOKE_PARAMETER_SETS` 存放常用参数组，`get_smoke_config()` 复用。
- `tester_*.py`：单策略冒烟入口  
  - 根据策略命名（`tester_null.py`、`tester_ga.py` 等）组织。  
  - 每个文件都提供 `PlacementStrategy` 实现、`run_xxx_strategy_test` 封装与 `smoke_test_xxx_strategy` 详细日志流程。
- `tester.py`：多策略比较实验控制器  
  - 解析 CLI 参数、构建 `TestConfig` 列表。  
  - `resolve_strategies()` 注册 `null/ga/gal/gal_pn/gal_sn/ft1/finetuned presets` 等策略。  
  - `Tester` 类按参数组循环执行并输出到 `tests/outs/tester_outs/`。

### 2. 使用方法

#### 单策略 smoke test
```bash
python tests/tester_null.py (--detail，已经默认启用)           # Null 策略
python tests/tester_ga.py --detail             # GA 策略（默认使用最新配置）
python tests/tester_gal.py --detail            # GAL
python tests/tester_ft1.py --detail            # FineTuned（需先准备模型）
```
- 所有 smoke 测试默认读取 `test_configs.get_smoke_config()`，自动在 `tests/outs/<scope>/session_*` 下生成日志与详细记录。
- 若策略需要额外参数（如 GA 的配置目录或 FineTuned 的模型路径），可通过各自 CLI 选项覆盖。

#### 多策略比较
```bash
python tester.py \
  --strategy null --strategy ga --strategy gal \
  --parameter "arrival_rate=0.5,mean_lifetime=30,max_time_steps=200,seed=1" \
  --parameter "arrival_rate=0.8,mean_lifetime=20,max_time_steps=200,seed=2"
```
- 不提供 `--parameter` 时使用顶层默认值（`--arrival-rate` 等）。
- `--workflow` 可指定自定义 VN 模板，`--disable-logging` / `--plot` 控制输出行为。

### 3. 策略实现思路速览

| 策略 | 文件 | 思路摘要 |
|------|------|----------|
| Null | `tests/tester_null.py` | 恒定拒绝所有任务，仅验证流水线；`place` 返回失败且不改动环境。 |
| GA | `tests/tester_ga.py` | 复制环境执行遗传算法 (`genetic_algorithm/ga_core.py`) 寻找映射；支持加载历史配置，返回最佳个体及 `fitness/path_length`。 |
| GAL | `tests/tester_gal.py` | 包装 `GAL.py` 中的贪心分配算法，在临时环境上运行原生 `GreedyAllocator`，返回映射。 |
| GAL-PN | `tests/tester_gal_2.py` | 逐节点贪心策略（Per-Node）：对VN节点按需求强度排序，每个VN节点选择后立即扣减资源，后续节点基于实时资源状态选择。 |
| GAL-SN | `tests/tester_gal_3.py` | SN节点预排序策略（SN-Sorted）：仅对SN节点按资源总量排序一次，VN节点按原始顺序依次从排序后的SN列表中选择。 |
| FineTuned (FT1) | `tests/tester_ft1.py` | 载入最新 `finetuning_putput/run_*` 模型，使用 `GreedyPPOAgent` 在环境副本上贪心推断；缺少模型时优雅跳过。 |
| 其他 FineTuned 预设 | `tests/tester_ft1.py` + `tester.py` | `get_finetuned_tester_registry()` 将 smoke 预设暴露给 `tester.py` 的对比实验。 |

#### 3.1 GAL 系列策略详细对比

三种 GAL 策略的核心区别在于**排序对象**和**资源扣减时机**：

**GAL（基础版本）**：
- **排序**：对VN节点按需求强度（cpu+mem+disk）从大到小排序；每次为VN节点选择时，对SN节点按剩余资源强度重新排序
- **选择策略**：为每个VN节点选择剩余资源最多的SN节点
- **资源扣减**：所有VN节点映射决策完成后，统一扣减资源
- **特点**：全局最优视角，但未考虑资源扣减对后续选择的影响

**GAL-PN（Per-Node，逐节点）**：
- **排序**：对VN节点按需求强度从大到小排序
- **选择策略**：为每个VN节点选择剩余资源最多的SN节点（基于当前实时资源状态）
- **资源扣减**：每个VN节点选择后**立即扣减资源**，后续节点看到的是已扣减后的资源状态
- **特点**：更贪心，每次选择都基于实时资源状态，能更好地适应资源变化

**GAL-SN（SN-Sorted，SN节点预排序）**：
- **排序**：仅对SN节点按资源总量（cpu+mem+disk）排序**一次**；VN节点不排序，按原始顺序处理
- **选择策略**：VN节点依次从排序后的SN节点列表中选择（从资源总量高的开始），找到第一个满足需求的SN节点即停止
- **资源扣减**：每个VN节点选择后立即扣减资源
- **特点**：计算开销最小（SN节点只排序一次），但选择策略较简单（不动态评估剩余资源）

所有策略都遵循：  
1. `prepare` 中处理自身依赖（加载模型、参数等）；  
2. `place` 仅返回映射，不直接扣减资源；  
3. `run_strategy_with_details` 统一调用 `_apply_mapping`、记录日志。

### 4. 扩展指引

1. 按 `tester_xxx.py` 模板新增策略，实现 `PlacementStrategy` 并提供 smoke CLI。  
2. 在 `test_printer.TEST_SCOPE_DIRS` 中登记新的 `*_single` scope。  
3. 如需参与 `tester.py` 对比实验，将策略工厂注册到 `resolve_strategies()`；若有多预设，可仿照 `get_finetuned_tester_registry()`。  
4. 更新 `策略接入规范.md` 与本文件，保持文档同步。
