## 策略测试接入规范

本文描述 `tests/` 目录中策略测试框架（`test_strategy.py` 等）的统一约定，供后续迁移 `tester4.py` 功能时参考。

### 1. 目录结构概览
- `test_strategy.py`：定义 `PlacementStrategy` 协议、`SingleTester` 执行单元以及详细日志运行函数。
- `test_printer.py`：负责输出目录寻址与 `DetailedTestLogger`。
- `test_configs.py`：集中存放 `TestConfig`、参数解析与常用参数组（如 `SMOKE_PARAMETER_SETS`）。
- `tester_*.py`：按策略划分的 smoke test 入口（Null/GAL/GAL2/GAL3/FT1 等），均依赖上述公共模块。
- `outs/`：`TestPrinter` 生成的输出目录，按 scope（`null_single_outs`、`gal_single_outs`、`tester_outs` 等）组织。

### 2. PlacementStrategy 接口约定
```python
class PlacementStrategy(Protocol):
    name: str
    def prepare(self, env: SimuVNEEnv) -> None: ...
    def place(self, vn: Data, sn_state: Data, env: SimuVNEEnv, *, context: StrategyContext) -> StrategyResult: ...
```

- **禁止**在 `place` 内直接修改 `env`（资源扣减、任务登记等副作用统一由框架完成）。
- 若策略需要试探扣减，必须在 `copy.deepcopy(env)` 上进行（参考 `GAL` 与 `FineTuned` 实现）。
- `StrategyResult` 应在失败时返回空映射并写入 `metadata["failure_reason"]` 便于日志定位。

### 3. 测试流程统一入口
- `SingleTester._dispatch_single_task` 与 `run_strategy_with_details` 在成功时唯一调用 `env._apply_mapping`，并将任务记录进 `env.active_workflows`。
- 详细日志通过 `TestPrinter.begin_step_logging → log_time_step → end_step_logging` 管理，每个 session 共用同一测试目录。
- 输出目录由 `TEST_SCOPE_DIRS` 控制，新增策略需为其 smoke 测试指定合适的 scope。

### 4. Smoke Test 模板
各 `tester_xxx.py` 应遵循以下结构：
1. 定义策略包装器（`PlacementStrategy` 实现），明确“只读环境”的 docstring。
2. 提供 `run_xxx_strategy_test`（封装 `run_single_strategy_test`）与 `smoke_test_xxx_strategy`（调用 `run_strategy_with_details`）。
3. `smoke_test` 默认使用 `get_smoke_config()`，在 `TestPrinter` 中设置对应 scope，并添加详细日志。
4. `__main__` 分支输出 summary 方便 CLI 验证。

### 5. 特殊策略
- **FineTuned** (`tests/tester_ft1.py`):
  - 默认查找 `/home/yc2/mrt/a/finetuning_putput` 下最新模型。
  - `place` 在深拷贝环境上运行 `PPOAgent`，只返回映射结果。
  - 若缺少模型文件，冒烟测试需优雅退出并提示路径信息。
- **GA** (`tests/tester_ga.py`):
  - 依赖 `genetic_algorithm/` 模块，可通过 `config_path` 或默认 `/home/yc2/mrt/a/ga_outputs` 的最新配置加载参数。
  - 进化过程在临时环境中读取资源，返回可行映射并提供 `fitness/path_length` 元数据。

### 6. 参数与比较实验
- `tester.py` 调用 `tests.test_configs` 中的 `ParameterSpec`、`build_round_configs` 解析参数组，并通过 `resolve_strategies` 注册 Null/GAL/GAL2/GAL3/FT1 及其预设。
- `main` 暴露 `--arrival-rate`、`--mean-lifetime`、`--max-time-steps`、`--seed`，未提供参数字符串时退化为单组实验。

### 7. 规范自检结论
截至本版：
- 所有策略均遵循“副作用隔离”原则，并在 docstring 中标明该约束。
- `TestPrinter` 使用 `outs/` 目录并提供详细日志共享机制，结构清晰。
- `test_configs.py` 已集中 smoke 参数，单策略冒烟测试全部接入。
- 资源扣减 Bug 已在 `test_strategy.py` 统一修复，`GAL`/`FT1` 等策略通过复制环境进行搜索，符合统一规范。

若后续新增策略或对比实验，请复用上述模板并更新本文件。
