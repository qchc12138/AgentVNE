## 一、核心功能模块

### 1. 模型架构 (`model.py`)

#### SimuVNE（策略网络）
```
输入: VN图 + SN图（6维特征，归一化）
  ↓
GNN编码（双GCN）
  ├─ VN: GCN → Self-Attention
  └─ SN: GCN → Self-Attention
  ↓
Neural Tensor Network (NTN)
  ↓
相似度矩阵 + 注意力矩阵
  ↓
MLP (256→128→N_s)
  ↓
输出: 概率矩阵 [N_v × N_s]
```

**特点**：
- GNN编码图结构
- 自注意力捕捉节点关系
- NTN计算节点对相似度

#### ValueNet（价值网络）
```
输入: VN图 + SN图
  ↓
双GCN编码
  ↓
图级池化
  ↓
拼接 [VN特征, SN特征]
  ↓
MLP (256→128→1)
  ↓
输出: 状态价值 V(s)
```

## 三、关键技术特性

### 1. 奖励设计（考虑资源竞争）

#### 即时奖励 r_t
```python
# 统计每条SN边被多少个workflow使用
for each workflow:
    for each VN_link:
        for each SN_edge in path:
            edge_usage_count[edge] += 1

# 计算加权跳数
for each workflow:
    hops = 0
    for each VN_link:
        for each SN_edge in path:
            k = edge_usage_count[edge]  # 共享次数
            hops += k  # 共享k次算k跳
    total_hops += hops

# 平均跳数（负值）
r_t = -total_hops / num_workflows
```

**设计意图**：
- 跳数越少越好
- 资源竞争越少越好
- r_t为负数，越接近0越好

#### 最终回报
```python
final_reward = T_total / T_p * sum(r_t)

其中:
  T_total = 总任务数
  T_p = 接受任务数
  sum(r_t) = 所有时刻的r_t之和
```

**设计意图**：
- 惩罚低接受率（T_total/T_p > 1）
- 惩罚高跳数（sum(r_t)为负）
- 鼓励高接受率 + 低跳数

### 2. 贪心放置调整（Fine-tuning）

在放置前，按VN需求优先级调整映射：

```python
# 1. 按需求排序VN节点
vn_nodes_sorted = sorted(vn_nodes, 
    key=lambda x: x.cpu_norm + x.mem_norm + x.disk_norm,
    reverse=True)

# 2. 对每个高需求节点，尝试重新映射
for vn_node in vn_nodes_sorted:
    if not can_fit(original_mapping[vn_node], vn_node):
        # 在所有SN中选择能容纳的最优节点
        new_sn = find_best_sn(vn_node)
        mapping[vn_node] = new_sn
```

**效果**：
- 优先保证高需求节点有资源
- 避免资源碎片化
- 提高放置成功率

## 四、性能对比

### GAL vs PPO（预期）

| 指标 | GAL（贪心） | PPO（神经网络） | 优势方 |
|------|------------|----------------|-------|
| **接受率** | 高（低负载） | 更高（学习优化） | PPO |
| **跳数** | 较高（局部最优） | 更低（全局优化） | PPO |
| **训练开销** | 无 | 有 | GAL |
| **推理速度** | 快 | 较慢 | GAL |
| **适应性** | 固定规则 | 可学习改进 | PPO |
| **可解释性** | 强 | 弱 | GAL |

### 测试场景建议

1. **低负载**（arrival_rate=0.2）
   - 预期：两者接受率都很高
   - 区别：跳数可能不同

2. **中负载**（arrival_rate=0.8）
   - 预期：PPO开始显示优势
   - GAL可能出现资源碎片化

3. **高负载**（arrival_rate=1.5）
   - 预期：PPO优势明显
   - GAL接受率显著下降

## 六、关键改进历史

### 1. 特征归一化（2025-11-03）
**问题**：价值损失很大（8000+），训练不稳定

**解决方案**：
- ✅ 统一归一化VN和SN特征到[0,1]
- ✅ 预训练和fine-tuning使用相同归一化
- ✅ 资源检查时反归一化

**效果**：
- 价值损失降低到合理范围
- 训练更稳定

### 2. 奖励反转（2025-11-03）
**问题**：r_t正负号导致优化方向错误

**解决方案**：
- ✅ r_t改为负值：`r_t = -avg_hops`
- ✅ final_reward去掉负号：`final_reward = T_total/T_p * sum(r_t)`

**效果**：
- 优化方向正确：最大化final_reward = 最小化跳数

### 3. 资源竞争考虑（2025-11-03）
**问题**：原r_t只统计物理跳数，未考虑带宽共享

**解决方案**：
- ✅ 如果SN边被k个workflow共享，算k跳
- ✅ 反映资源竞争的真实代价

**效果**：
- 奖励更准确
- 鼓励避免资源冲突

### 4. 贪心放置调整（2025-11-04）
**问题**：神经网络采样的映射可能不可行

**解决方案**：
- ✅ 按VN需求优先级排序
- ✅ 重新分配不可行的映射

**效果**：
- 提高放置成功率

### 5. GAL对比算法（2025-11-04）
**需求**：需要baseline评估PPO性能

**解决方案**：
- ✅ 实现贪心放置算法
- ✅ 使用完全相同的环境
- ✅ 唯一差异是放置策略

**效果**：
- 公平的性能对比基准
