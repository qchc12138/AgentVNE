# 节点排序问题修复说明

## 问题诊断

原始代码存在以下问题：

### 1. 不必要的节点排序
- `_topology_to_pyg_data()` 函数对节点按ID进行排序
- 这在图神经网络中是**不必要的**，因为：
  - GNN通过 `edge_index` 定义节点间的连接关系
  - 节点顺序不影响图结构的表达
  - 排序反而增加了复杂性

### 2. 索引映射的复杂性
原代码维护了多个索引映射：
- `node_id_to_new_idx`: 节点ID到排序后索引
- `original_idx_to_new_idx`: 原始索引到排序后索引
- 需要 `_reorder_array_by_index_mapping()` 来重排数组

这导致：
- 代码逻辑复杂，难以维护
- 容易出现索引不一致的bug
- 性能开销（额外的排序和映射操作）

### 3. 潜在的不一致性
- `_greedy_place_workflow()` 使用原始拓扑和原始节点索引
- 但图数据（workflow_graph, substrate_graph）使用排序后的节点顺序
- 标签 `y` 的维度对应排序后的节点
- 需要在测试打印时进行索引转换，容易出错

## 修复方案

### 修改1: 简化 `_topology_to_pyg_data()`
```python
# 修改前：
def _topology_to_pyg_data(..., return_index_mapping: bool = False):
    # 对节点排序
    nodes_with_idx.sort(...)
    # 创建映射
    node_id_to_new_idx = {}
    original_idx_to_new_idx = {}
    # 更新边的索引
    ...
    if return_index_mapping:
        return data_obj, original_idx_to_new_idx
    return data_obj

# 修改后：
def _topology_to_pyg_data(...):
    # 直接使用原始节点顺序，无需排序
    x = _nodes_to_features(nodes, ...)
    edge_index = _build_edge_index(links, directed)
    return Data(x=x, edge_index=edge_index)
```

### 修改2: 删除 `_reorder_array_by_index_mapping()`
这个函数不再需要，因为节点顺序保持一致。

### 修改3: 移除所有索引映射相关代码
- 删除 `return_index_mapping` 参数
- 删除 `sn_idx_mapping` 变量
- 删除对 `_reorder_array_by_index_mapping()` 的调用
- 简化测试打印代码（直接使用原始索引）

## 修改效果

### 优点
1. **代码简化**：删除约40行冗余代码
2. **逻辑清晰**：所有操作基于统一的节点顺序
3. **减少bug**：消除索引不一致的风险
4. **性能提升**：避免排序和映射开销
5. **易于维护**：减少心智负担

### 保持不变
- 数据集的语义完全不变
- 模型输入输出维度不变
- 训练和验证流程不变
- 生成的标签含义不变

### 兼容性
- **需要重新生成数据集**：因为节点顺序发生变化
- 旧的预训练模型需要重新训练
- 但这是一次性的迁移成本，换来长期的代码质量提升

## 验证要点

1. **节点顺序一致性**：
   - `sn_noderank` 的索引应与 `substrate_graph.x` 的行索引一致
   - `workflow_noderank` 的索引应与 `workflow_graph.x` 的行索引一致
   - 标签 `y[i, j]` 对应 VN节点i 到 SN节点j 的分配概率

2. **边索引正确性**：
   - `edge_index` 中的节点索引应该是列表索引（0, 1, 2, ...）
   - 确保 links 中的 source/target 是索引而非节点ID

3. **放置映射一致性**：
   - `_greedy_place_workflow()` 返回的映射使用原始索引
   - 与图数据的节点索引直接对应

## 后续建议

如果拓扑文件中的 links 使用节点ID而非索引，建议在 `_build_edge_index()` 中添加ID到索引的映射逻辑，而不是在 `_topology_to_pyg_data()` 中处理。
